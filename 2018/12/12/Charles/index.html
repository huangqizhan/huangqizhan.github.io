Charles 使用说明

  • charles作为客户端与服务端的一个中间代理 即充当客户端的服务器 充当服务端的客户端
  • 在同一个局域网之下设置客户端的代理为charles ip地址和端口号
  • 一般情况下客户端发起请求经过charles代理 再有charles跟服务器交互
  • 服务器端返回经过charles 再有charles到客户端

iOS SSL证书配置

  • 设置好客户端代理之后 在fafari 输入 chal.pro/lls 安装证书, 安装之后 还需要信任 设置 -> 通用 -> 关于本机 -> 证书信任设置
  • 在设置里面信任证书 这样可以充当客户端的CA证书 保证验证通过
  • Mac下安装SSL证书 help -> SSL Proxying -> install Charles Root Certificates 这样可以充当服务端的CA证书
  • 在钥匙串中双击 SSL 信任 最后保存

如果HTTPS客户算没有SSL验证 可以避免乱码

  • 上方菜单栏 —-> Proxy —-> SSL Proxy Settings —-> Add
  • Host：填*表示所有网站都抓
  • Port：443

自定义客户端请求的数据

  • 右键单击 Structure -> Map location
  • 可以指定自定义的数据文件
  • 下次请求可获取到自定义的数据文件
